## はじめに

2026年6月18日、PDF Association から **PDF 2.0 Errata Collection 3（EC3）** が公開されました。

https://pdfa.org/pdf-2-0-errata-collection-3-now-available/

PDFの現行仕様である ISO 32000-2:2020（PDF 2.0）は、本来は有償のISO規格ですが、Adobe・Apryse・Foxit のスポンサードにより2023年4月から**無償で入手できます**。EC3 は、この無償版に適用される正誤表コレクションの第3版です。

この件、英語圏でもアナウンス以上の解説は少なく、日本語の情報はほぼ皆無です。筆者は PDF 仕様書を参照する MCP サーバー（[pdf-spec-mcp](https://github.com/shuji-bonji/pdf-spec-mcp)）を開発しており、EC3 対応のために中身を検証したので、その結果も踏まえて紹介します。

### この記事で扱うこと

- Errata Collection という仕組み（ISOの正誤表がどう管理・配布されているか）
- EC3 の内容 — 356件の修正、承認率、追補20ページ
- EC2 から実際に変わったファイルの検証結果
- 入手方法と、読むときの注意点（**ビューア選びが重要**）

## Errata Collection という仕組み

「ISO規格の正誤表」と聞くと、ISOが発行する Technical Corrigendum（技術的正誤票）を想像するかもしれません。PDF 2.0 の場合は少し違う、面白い運用がされています。

errata の受付は PDF Association の **公開 GitHub リポジトリ** で行われています。誰でも Issue を立てられます。

https://github.com/pdf-association/pdf-issues

提起された errata は2つの承認経路で処理されます。

| 承認主体                                | ステータス表記 | 意味                        |
| --------------------------------------- | -------------- | --------------------------- |
| ISO TC 171 SC 2 WG 8（ISO 32000担当WG） | Completed      | ISO承認済み                 |
| PDF Association 技術WG（PDF TWG）       | Accepted       | 業界承認済み（ISO承認待ち） |

承認された修正は、無償版PDFに**注釈（アノテーション）として直接適用**され、「Errata Collection」として定期的にまとめて公開されます。紙の正誤表を別紙で配るのではなく、仕様書PDF自体に朱入れして配る方式です。解決済み errata の全リストは [pdf-issues.pdfa.org](https://pdf-issues.pdfa.org/) で確認できます。

## EC3 の内容

EC3 は EC2 の**完全な置き換え**です（差分パッチではありません）。概要は以下のとおりです。

| 項目              | EC3                                                              | 参考: EC2 |
| ----------------- | ---------------------------------------------------------------- | --------- |
| 基準日            | 2026年6月1日                                                     | 2023年    |
| errata 修正数     | **356件**（766編集）                                             | —         |
| ISO承認済みの編集 | 655 / 766（**86%**）                                             | 54%       |
| 追補ページ        | **20ページ**                                                     | 17ページ  |
| 対象の最新 Issue  | [#742](https://github.com/pdf-association/pdf-issues/issues/742) | —         |

注目は承認率です。EC2 時点で業界承認どまりだった多数の errata が、EC3 では ISO 承認に格上げされています。「GitHubで受け付けた修正が ISO の正式な承認プロセスに載っていく」流れが実際に機能していることが分かります。

### 修正はどう実装されているか

766編集の内訳は次のとおりです。

| 実装方法                       | 件数 |
| ------------------------------ | ---- |
| テキスト置換注釈               | 322  |
| テキスト挿入注釈（キャレット） | 200  |
| 付箋（sticky note）            | 124  |
| 取り消し線（削除）注釈         | 113  |
| ファイル添付                   | 6    |

図や表の差し替えなど注釈で表現しきれない修正は、**巻末に追補ページとして追加**されています（EC3で20ページ。ページラベルは「Errata-1」〜「Errata-20」）。各追補ページには元の位置との間を行き来するナビゲーションボタンが付いています。

各注釈には対応する GitHub Issue 番号が記載されており、修正の背景となった議論まで遡れます。

### 一部は ISO 32000-2 の正式な改訂（Amendment）へ

EC3 に含まれる errata のうち一部は、今後発行される ISO 32000-2:2020 への ISO amendment にも収録される予定です。対象は ISO/TS 32005（構造要素の名前空間マッピング）が必要とする修正で、これは [WTPDF](https://pdfa.org/wtpdf/)・PDF/UA-2（ISO 14289-2）、そして今後の PDF/A-4（ISO 19005-4）・PDF/X-6（ISO 15930-9）の日付付き改訂版にも波及します。アクセシビリティ系の仕様が PDF 2.0 本体の修正を要求している構図です。

## EC2 から実際に変わったファイルを検証する

無償バンドルには ISO 32000-2 本体のほか、TS文書やアプリケーションノートなど計11ファイルが含まれます。EC3 公開に伴い再ダウンロードした全ファイルを、手元の EC2 世代とハッシュ・メタデータ・ページ数で比較しました。

| ファイル                               | 変更 | 詳細                                          |
| -------------------------------------- | ---- | --------------------------------------------- |
| ISO 32000-2:2020                       | ✅   | 1,020頁 → **1,023頁**（追補17→20ページ）      |
| ISO/TS 32001:2022（SHA-3ハッシュ拡張） | ✅   | 13頁 → 14頁、タイトルに "Errata Collection 3" |
| ISO/TS 32002:2022（ECC/PAdES署名拡張） | ✅   | 13頁 → 14頁、同上                             |
| ISO/TS 32003 / 32004 / 32005           | ❌   | 内容同一                                      |
| PDF-Declarations、AN001〜003、WTPDF    | ❌   | ハッシュ完全一致                              |

実質の更新は **3ファイルだけ** です。

一点、注意があります。TS 32003〜32005 は「内容同一」なのに**ファイルサイズとハッシュは旧版と一致しません**。スポンサー版PDFはダウンロード時に利用者名・所属のスタンプが埋め込まれるため、バイト列が毎回変わるのです。ハッシュ比較だけで「更新された」と判断すると誤ります（筆者は一瞬騙されました）。ページ数・タイトル・ModDate を併せて確認し、公式アナウンスの "Other sponsored ISO publications are unchanged since Errata Collection 2" という記述と突き合わせて確認しました。

## 入手方法

以下から無償でダウンロードできます。

https://pdfa.org/sponsored-standards/

ISO・ANSI との契約上、各コピーには利用者の氏名・所属がスタンプされる仕組みで、**EC2 を取得済みの人も再登録（再ダウンロード）が必要**です。登録情報が ISO や第三者に提供されることはない、と明記されています。

## 読むときの注意：ビューア選びが重要

前述のとおり、errata は PDF 注釈として実装されています。つまり——

**注釈表示に対応していないビューアで開くと、errata 修正の存在に気づかないまま「修正前の仕様」を読んでしまいます。**

これはかなり実害のある罠です。最低限、マークアップ注釈とファイル添付注釈に対応したビューア（Adobe Acrobat Reader、Apryse、Foxit 等）を使い、必要に応じてコメントパネルを開いてください。ブラウザ内蔵ビューアやテキスト抽出ツールでは注釈が見えないことがあります。

なお、注釈にはステータス値が設定されており、承認主体を見分けられます。

- **Completed** = ISO TC 171 SC 2 WG 8 承認（ISO承認済み）
- **Accepted** = PDF TWG 承認（業界承認）

ビューアによってアイコンや色の表現は異なります。

### ページラベルも改善されている

EC2 以降、PDF のページラベル（ISO 32000-2, 12.4.2）が ISO のページ表記と一致するよう修正されています。

| 部分                        | ページラベル                  |
| --------------------------- | ----------------------------- |
| PDF Association の表紙      | Cover-A、Cover-B              |
| ISO 前付け（Foreword等）    | i 〜 xiii（小文字ローマ数字） |
| 本文（Clause 1 Scope 以降） | 1 〜 1,003                    |
| EC3 追補ページ              | Errata-1 〜 Errata-20         |

対応ビューアなら「ページ 55」と入力すると ISO のフッター表記どおりのページに飛べます。物理ページ番号とのズレに悩まされてきた人には地味に嬉しい改善です。

### 機械可読な PDF 定義も更新されている

errata を反映した機械可読の PDF 定義として、[Arlington PDF Model](https://github.com/pdf-association/arlington-pdf-model) も更新されています。PDF パーサーやバリデータを実装する場合は、こちらが最新の定義になります。

## まとめ

- EC3 は PDF 2.0 無償版の正誤表コレクション第3版。**EC2 の完全置き換え**で、356件（766編集）の修正と追補20ページを含む
- ISO承認率が 54% → **86%** に向上。GitHub発の errata が ISO の正式プロセスに載る運用が機能している
- 実際に更新されたのは ISO 32000-2 本体と TS 32001/32002 の**3ファイルのみ**。ただしスポンサー版はDLスタンプでハッシュが毎回変わるため、バイト比較では内容差分を判定できない
- errata は**注釈実装**。注釈対応ビューアで読まないと修正の存在自体に気づけない
- 入手には再登録が必要（無料）

PDF 2.0 を実装・参照する開発をしているなら、手元の仕様書を EC3 に差し替えておくことをおすすめします。
